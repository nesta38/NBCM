Comment ça fonctionne ?
  1. Configurer l'API : Assurez-vous que l'API est configurée avec l'URL et le token dans Configuration API
  2. Définir l'intervalle : Choisissez la fréquence d'import (ex: 60 minutes = 1 fois par heure)
  3. Activer la planification : Cochez la case "Planification active"
  4. Redémarrer l'application : Exécutez docker-compose restart nbcm pour appliquer les changements
  5. Vérifier les logs : Consultez les logs pour voir les imports automatiques : docker-compose logs -f nbcm-app | grep API
  6. Historique : Consultez l'historique des imports pour voir les résultats
Intervalles recommandés
  • 15-30 min : Environnements avec backups fréquents
  • 60 min (1h) : Usage standard recommandé
  • 2-6h : Environnements peu actifs
  • 12-24h : Imports ponctuels
Limitations
  • Minimum : 5 minutes (éviter la surcharge)
  • Maximum : 1440 minutes (24h)
  • L'API Veritas peut avoir des limites de taux
  • Redémarrage requis après modification
Dépannage
  • Vérifier les logs d'import
  • Tester manuellement dans Config API
  • Vérifier l'URL et le token
  • Consulter les logs Docker : docker-compose logs nbcm-app
